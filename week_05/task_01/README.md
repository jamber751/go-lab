# Неделя 5, Задание 1: Программирование алгоритма с вложенными циклами

## Описание задания

**Задача:** Найти первые 100 простых чисел, используя алгоритм с вложенными циклами.

Простое число — это натуральное число, большее 1, которое имеет только два делителя: 1 и само себя.

## Требования

1. Реализовать функцию проверки числа на простоту с использованием вложенных циклов
2. Найти первые 100 простых чисел
3. Вывести результат в удобном формате (в колонках)
4. Добавить проверку корректности алгоритма
5. Продемонстрировать различные варианты использования вложенных циклов

## Алгоритм

### Основная логика поиска простых чисел:

1. **Функция `isPrime(n)`** - проверяет, является ли число простым:
   - Исключаем числа ≤ 1 (не простые)
   - 2 - единственное четное простое число
   - Проверяем четность (четные числа > 2 не простые)
   - Используем вложенный цикл для проверки делителей от 3 до √n с шагом 2

2. **Функция `findFirstNPrimes(n)`** - находит первые n простых чисел:
   - Внешний цикл продолжается, пока не найдем n простых чисел
   - Внутренняя проверка каждого числа на простоту

3. **Функция `printPrimesInColumns(primes, columns)`** - форматированный вывод:
   - Внешний цикл для строк
   - Внутренний цикл для колонок в каждой строке

## Особенности реализации

### Использование вложенных циклов:

1. **В функции `isPrime`:**
   ```go
   for i := 3; i <= int(math.Sqrt(float64(n))); i += 2 {
       if n%i == 0 {
           return false
       }
   }
   ```

2. **В функции `findFirstNPrimes`:**
   ```go
   for len(primes) < n {        // Внешний цикл
       if isPrime(num) {        // Внутренняя проверка
           primes = append(primes, num)
       }
       num++
   }
   ```

3. **В функции `printPrimesInColumns`:**
   ```go
   for i := 0; i < len(primes); i += columns {     // Внешний цикл для строк
       for j := 0; j < columns && i+j < len(primes); j++ {  // Внутренний цикл для колонок
           fmt.Printf("%4d ", primes[i+j])
       }
       fmt.Println()
   }
   ```

## Оптимизации

1. **Проверка только до √n:** Достаточно проверить делители до квадратного корня из числа
2. **Проверка только нечетных делителей:** После проверки на четность проверяем только нечетные числа
3. **Ранний выход:** Как только найден делитель, функция сразу возвращает false

## Сложность алгоритма

- **Временная сложность:** O(n√n) для поиска n простых чисел
- **Пространственная сложность:** O(n) для хранения найденных простых чисел
