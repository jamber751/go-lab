# Неделя 5 - Вложенные циклы и алгоритмы

## Обзор недели

Пятая неделя посвящена изучению вложенных циклов в Go и их применению для решения алгоритмических задач. Студенты изучают эффективные алгоритмы поиска простых чисел и учатся оптимизировать производительность программ.

## Задания

### [Задание 1: Программирование алгоритма с вложенными циклами](./task_01/)

**Описание:** Программа для поиска первых 100 простых чисел с использованием вложенных циклов и различных алгоритмических оптимизаций.

**Ключевые темы:**
- Вложенные циклы в Go
- Алгоритмы поиска простых чисел
- Оптимизация производительности
- Математические функции (`math.Sqrt`)
- Форматированный вывод результатов

**Алгоритм:**
- Функция `isPrime(n)` - проверка числа на простоту
- Функция `findFirstNPrimes(n)` - поиск первых n простых чисел
- Функция `printPrimesInColumns()` - форматированный вывод

**Файлы:**
- [main.go](./task_01/main.go) - исходный код
- [README.md](./task_01/README.md) - подробное описание задания
- [go.mod](./task_01/go.mod) - модуль Go

**Запуск:**
```bash
cd week_05/task_01
go run main.go
```

## Технические навыки

### Вложенные циклы
- Синтаксис вложенных циклов `for` в Go
- Применение вложенных циклов для поиска делителей
- Оптимизация вложенных циклов

### Алгоритмические оптимизации
- Проверка делителей только до √n
- Пропуск четных чисел после проверки на четность
- Ранний выход из циклов при нахождении результата

### Математические функции
- `math.Sqrt()` для вычисления квадратного корня

### Форматирование вывода
- Вывод чисел в колонках
- Контроль ширины и выравнивания
- Структурированный вывод результатов

## Особенности реализации

### Алгоритм поиска простых чисел
1. **Исключение тривиальных случаев:** числа ≤ 1 не являются простыми
2. **Проверка четности:** 2 - единственное четное простое число
3. **Оптимизированная проверка делителей:** проверяем только нечетные числа до √n
4. **Ранний выход:** прекращаем проверку при нахождении первого делителя

### Структура программы
- **Модульность:** каждая функция решает конкретную задачу
- **Читаемость:** подробные комментарии на русском языке
- **Тестирование:** проверка корректности на известных простых числах
- **Статистика:** вывод информации о найденных числах

### Производительность
- **Временная сложность:** O(n√n) для поиска n простых чисел
- **Пространственная сложность:** O(n) для хранения результатов
- **Оптимизации:** минимизация количества проверок
